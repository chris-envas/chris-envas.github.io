关于this的指向

发表于 2018-12-08 | 分类于 前端

this 关键字是JavaScript中最复杂的机制。它是一个很特别的关键字，被自动定义在所有函数的作用域中，但是即使是非常有经验的JavaScript开发者也很难说清它到底指向什么
ES6的胖箭头函数(=>)，不过是对ES5的this规则的封装利用，ES6给我们提供了更强的API和语法糖,但ES5是JS的核心基础，深刻的掌握this,可以为你的ES6学习提供更强的助力
开始吧
先记住一句话 this实际上是在函数被调用时发生的绑定，它指向什么完全取决于函数在哪里被调用！

误解1 this指向自身

许多人,认为this是指向函数自身，从英语语法的角度来讲是说得通，但事实并非如此

误解2 this指向函数的作用域

在任何情况下，this都不指向函数的词法作用域，尽管作用域与对象类似，可见的标识符也是它的属性，但作用域无法通过JS代码访问，它存在于JS引擎内部

关于this

  1. this是在运行时绑定，而不是在编写时绑定，它的上下文取决于函数调用的各种条件。
  2. this的绑定和函数声明的位置无关，只取决于调用的方式
  3. this被调用时会创建一份活动记录（执行上下文），这个记录包含函数被调用时的所有信息，而this就是被记录的其中一个属性，会在执行函数的过程用到

全面解析this

简单的讲,this是在运行时被绑定的，取决于函数调用位置，只有仔细的分析调用的位置，才能准确的知道this到底引用的是什么

找到了调用位置 我们就只需要根据四条规则来判断this的指向

默认绑定

解析： 你是否一眼就判断出foo函数的调用位置是全局作用域，并且foo()是直接使用而且不带任何修饰的函数进行调用的，没错啦，当函数不携带任何的修饰函数直接调用时，this指向的就是全局window（严格模式下为undefind）,这就是默认绑定规则！

隐式绑定

解析： 隐式规则会判断调用位置是否存在上下文，如果存在隐式规则会把函数中的this绑定到上下文对象，因此foo()被调用时，this被绑定到了obj对象上，所以this.a和obj.a是一样的

显示绑定

隐式绑定时,我们必须在一个对象内部包含一个指向函数的属性，通过属性的间接引用函数，从而把this隐式的绑定在该对象上。
如果我们不愿采用这种方式，而想在某个对象上强制调用函数，该怎么办？答案很简单，想必你也曾经使用过，那就是call() apply()方法(当然，还有一个叫bind()的玩意)

解析： 显示绑定可以硬绑定this的指向，我们创建了bar()函数，内部手动的调用foo.apply(obj),强制绑定了this的指向，这便是显示绑定中的-硬绑定

显示绑定-硬绑定的典型应用场景就是，创建一个包裹函数没传入所有参数，并返回接收所有的参数

另一种使用方法就是创建可以重复使用的辅助函数

API调用“上下文”，JS中存在许多内置函数，都提供了一个可选参数，通常成为上下文(context)，其作用与bind()一样，底层也是视同call() apply()实现的

new绑定

如果你缺一个女朋友，是不是new一个就行了
在传统的面向类语言中，使用new调用的函数我们称之为“构造函数”，它是类中的一个特殊的方法，很遗憾的是，实际上JS并不存在“构造函数”(虽然我们常常会讲构造函数，但只不过是我们口头上遵照的一种标准)，当我们使用new调用某个函数时，只是对于函数进行“构造调用”，所以一切都得靠我们自己，我相信这让我们更接近底层，似乎不是件坏事
JS的new机制与其他面向类语言不一致，那么它究竟做了什么？

  1. 创建（构造）了一个全新的对象。
  2. 这个新对象会被执行[[prototype]]连接
  3. 新对象会绑定到函数调用的this上
  4. 如果函数没有返回其他对象，那么new表达式中的函数会自动返回这个新对象

优先级

new绑定和显示绑定 都大于隐式绑定 那么new绑定和显示绑定谁的优先级更高呢?

解析： 简单的讲,使用了bind()函数的显示绑定(硬绑定)会判断是否被new 调用,如果是，则会更改this的绑定,由此可见new绑定优先级大于显示绑定

底层参考MDN关于bind的polyfill(https://developer.mozilla.org/zhCN/docs/Web/JavaScript/Reference/Global_Objects/Function/bind)

同时使用了new绑定和显示绑定(硬绑定)，可以为预设的函数先设置参数，这样在new初始化时，只需传递剩余的参数。
bind()函数的第一个参数用于绑定this，其余参数则可以传递给下层的函数，正是利用了这点
把多参数的函数转为单参数函数就是“柯里化”
阮一峰函数柯里化(http://www.ruanyifeng.com/blog/2017/02/fp-tutorial.html)

判断this

总结完四条规则，我们就可以根据优先级来判断函数在某个调用位置应用的是哪条规则

  1. 函数是否存在new绑定 如果是this绑定的将是新创建的对象 eg: var bar = new foo()
  2. 函数是否存在call apply显示绑定 如果是this绑定的将是指定对象 eg: var bar = foo.call(obj)
  3. 函数是否在某个上下文对象中被调用（隐式绑定）如果是this绑定的将是该上下文对象 eg: var bar = obj.foo()
  4. 以上情况均不存在,默认绑定window,严格模式下undefind
