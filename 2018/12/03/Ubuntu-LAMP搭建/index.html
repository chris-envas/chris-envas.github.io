Ubuntu-LAMP搭建

发表于 2018-12-03 | 更新于 2019-07-26 | 分类于 服务器

快速搭建LAMP环境,小白进击Linux
类似 LAMP 等的 Linux 经典服务器部署，可以查找这2家VPS供应商的参考文档。
https://www.digitalocean.com/、 https://www.linode.com

  • 为了少打sudo，加快部署，直接sudo su

  • 更新系统软件源，的软件包（二进制包）列表.
    apt update

  • 安装Apache
    apt install apache2

  • 开启 Apache2 的 KeepAlive
    vi /etc/apache2/apache2.conf

  • 安装Mysql
    apt install mysql-server
    安装过程中会提示你输入root密码
    如果不幸MySQL的root密码忘记了，谷歌搜索：mysql_secure_installation 相关的文章，会教你怎样重置 root密码

  • 安装PHP7
    apt install php libapache2-mod-php php-mysql

  • 全局 加入 index.php 为默认索引文件（不需要全局的话，可以忽略这部，后面在 vhost的网站配置里改）
    vi /etc/apache2/mods-enabled/dir.conf
    索引类型增加 index.php ，注意插入位置，关系到优先级（同目录下，这些名字的文件的优先级别）

  • 重启Apache2 ( PHP 是Apache2 的模块，所以PHP 配置修改了，也必须执行重启Apache2)
    systemctl restart apache2

  • 查询可更新的 其他 php模块
    apt search php7.0-

#############

  • 这个时候，LAMP已经装好了，我们的环境如下
    Default host www-root: /var/www/html/
    apache2环境的php配置文件: /etc/php/7.0/apache2/php.ini
    命令行下起作用的php配置文件: /etc/php/7.0/cli/php.ini

我们主要操作 -available/ 目录下的 .conf
然后再用 a2enconf a2enmod a2ensite 这3个工具启用你配置的 .conf
然后再用 a2disconf a2dismod a2dissite 这3个工具禁用你配置的 .conf

  • /etc/apache2/mods-available/
    先说 mod （模块）， 大部分 Apache2 的动态库都是装了的，这些都是功能模块，但是 为了安全、轻量 等原因，这些模块是没有启用的

  • /etc/apache2/conf-available/
    全局的HTTP 协议的一些参数，另外一些全局参数，是在上面的 mods 里管理的

  • /etc/apache2/sites-enabled
    上面两个东西其实需要的机会不多，最经常操作的还是 vhost （sites）
    /etc/apache2/sites-available/{000-default.conf,default-ssl.conf}
    分别是 80(http://) 和 443(https://) 2个 默认端口服务的 默认网站设置，
    默认网站就是：
    ​ 当你的客户端请求头文件，没有指定 Host: xx.com 来访问这个服务器的时候，或者是，Host: xx.com 在Apache的sites里根本不存在，就会使用这个默认网站的内容，返回给用户请求
    ​ 当你直接用 IP 打开的时候，也是访问这个默认网站

  • 创建一个 xx.com 的 vhost

a2ensite xx.com; systemctl reload apache2
mkdir -p /var/www/xx.com
chown -R www-data:www-data /var/www/xx.com # php 要读写web目录的时候，需要如此给予权限

  • 第二个vhost 测试

mkdir -p /var/www/bbs.xx.com
chown -R www-data:www-data /var/www/bbs.xx.com
a2ensite bbs.xx.com; systemctl reload apache2

a2ensite bbs.xx.com # 启用这个网站配置
a2dissite bbs.xx.com # 禁用这个网站配置
