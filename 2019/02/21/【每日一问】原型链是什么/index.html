【每日一问】原型链是什么

发表于 2019-02-21 | 更新于 2020-06-20 | 分类于 前端

JS是一门基于原型的语言，使用面向对象(oop)的方式来开发时，常常会碰到继承的问题。理解原型链的工作原理，势必事半功倍。

什么是原型链

我们并未对obj赋予任何的属性和方法

可是它为什么有 valueOf / toString/constructor 属性呢？使用console.dir()打印结果如下

又上图可知，obj对象有一个__proto__属性，而obj.__proto__又包含了 valueOf / toString/constructor三个属性

由此可见，似乎当我们调用obj.valueOf()时，JS引擎似乎会在检查obj对象不存在valueOf()后，沿着obj.__proto__继续寻找，直至找到对应的方法为止

这个设想是成立的，JS引擎确实是这样的运行的，这一切都是因为JS是一门基于原型的语言

JS引擎沿着__proto__不断搜索的这个过程，构造了一条由__proto__组成的原型链子我们称之为原型链

共享原型链

原型链有一大特点就是共享（对，就像共享单车，大家都可以骑..emm）

在文章开头我们创建了obj对象，在那之后，我们没有对其做过任何处理，它天生就拥有了一个__proto__属性，现在我们再来创建一个obj2对象,额，它也天生拥有一个__proto__属性，接着我们来做一个有趣的比较

二者的__proto__是相等的，这就是共享原型链！
